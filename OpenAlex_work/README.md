# базовый рабочий процесс

это Python-проект для получения данных о научной продукции университетов из **OpenAlex**.

Эта версия рабочего процесса описывает только **базовый метод**: быстрый и прямой способ поиска университетов, получения их институциональных метаданных и экспорта исследовательских показателей в CSV.

---

## Назначение

Базовый рабочий процесс предназначен для быстрого и простого извлечения данных о количестве научных работ университетов из OpenAlex.

Он полезен, если вы хотите:

- быстро протестировать проект
- обработать небольшой или средний список университетов
- получить CSV-вывод с минимальной настройкой
- сделать proof of concept перед созданием более продвинутого конвейера

---

## Требования

Перед запуском скрипта убедитесь, что у вас есть:

- Python 3.12 или новее
- [`uv`](https://docs.astral.sh/uv/) для управления зависимостями и окружением
- API-ключ OpenAlex
- доступ в интернет
- файл `.env` с вашими учетными данными

Пример файла `.env`:

```env
OPENALEX_API_KEY=your_openalex_api_key
OPENALEX_MAILTO=you@example.com
```

Установите зависимости командой:

```bash
uv sync
```

Или установите их вручную:

```bash
uv add httpx pandas tenacity rapidfuzz python-dotenv
```

## Базовый скрипт

```text
src/basic/fetch_university_counts.py
```

Этот скрипт читает список названий университетов, выполняет прямой поиск в OpenAlex, выбирает наиболее подходящее совпадение институции, получает связанные метаданные и записывает результаты в CSV-файл.

---

## Входной файл

Подготовьте CSV-файл, содержащий названия университетов.

Рекомендуемый формат:

```csv
university_name,country_code
South Ural State University,RU
Moscow State University,RU
Saint Petersburg State University,RU
```

### Обязательные столбцы

- `university_name`

### Рекомендуемые столбцы

- `country_code` — настоятельно рекомендуется для повышения точности сопоставления

---

## Что делает скрипт

Скрипт выполняет следующие шаги:

1. читает входной CSV-файл
2. ищет каждый университет по названию в OpenAlex
3. выбирает наилучшее доступное совпадение институции
4. получает метаданные институции
5. извлекает исследовательские показатели
6. экспортирует результаты в выходной CSV-файл

---

## Как запустить

Пример с явным указанием путей входного и выходного файлов:

```bash
uv run src/basic/fetch_university_counts.py --input data/universities.csv --output data/results.csv
```

Если в скрипте уже настроены пути по умолчанию:

```bash
uv run src/basic/fetch_university_counts.py
```

---

## Ожидаемый результат

Скрипт создает CSV-файл, содержащий исследовательские показатели и связанные метаданные институции.

Типичные поля выходного файла:

- исходное название университета
- код страны
- сопоставленное название институции в OpenAlex
- идентификатор институции OpenAlex
- идентификатор ROR
- количество работ
- количество цитирований
- домашняя страница институции
- статус сопоставления

Пример:

```csv
query_name,country_code,matched_name,openalex_id,works_count,cited_by_count,match_status
Moscow State University,RU,Lomonosov Moscow State University,I19880235,200147,8771838,matched
```

---

## Ограничения

У базового рабочего процесса есть и ограничения:

- сопоставление выполняется напрямую по названиям
- неоднозначные названия университетов могут приводить к неправильным совпадениям
- при использовании в большом масштабе может потребоваться ручная проверка

---

## Когда использовать базовый рабочий процесс

Используйте этот метод, если:

- вы все еще тестируете идею
- вам нужен максимально короткий путь к результату
- ваш входной список небольшой или средний
- допустима умеренная точность
- вы готовы вручную проверить несколько неоднозначных совпадений
